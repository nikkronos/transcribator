# Спецификация: MVP транскрибатора

## Что надо сделать

Реализовать локальную CLI-утилиту, которая принимает один или несколько локальных аудио/видео файлов, распознаёт речь (русский) и сохраняет результат в виде обычного текста (.txt) и структурированного JSON с таймкодами (.json). Утилита должна работать офлайн.

## Как это должно работать

- Пользователь запускает утилиту из терминала, передавая пути к файлам (и опционально папку для результатов).
- Для каждого файла:
  - из видео при необходимости извлекается аудиодорожка (через ffmpeg);
  - аудио передаётся в локальную модель распознавания речи (faster-whisper);
  - генерируются два файла рядом с исходным (или в указанной папке): `имя_файла.txt` и `имя_файла.json`.
- .txt — один текст без разметки (все сегменты подряд).
- .json — структура с сегментами: массив объектов с полями `start`, `end` (секунды), `text` (и при необходимости `language`), плюс метаданные (имя файла, модель, язык).
- Поддерживаются форматы: аудио (mp3, wav, m4a, ogg, flac и т.д.) и видео (mp4, avi, mkv и т.д.) — всё, что можно преобразовать в аудио через ffmpeg.
- Можно указать несколько файлов за один запуск; они обрабатываются последовательно.
- Язык распознавания: русский (фиксировано в MVP).

## Инструменты

- **Python 3.10+**
- **faster-whisper** — локальное распознавание речи (CTranslate2, поддержка русского, сегменты с временами).
- **ffmpeg** — извлечение аудио из видео и при необходимости конвертация в формат для whisper (устанавливается отдельно на систему или через путь).
- **requirements.txt** — faster-whisper, при необходимости вспомогательные библиотеки (например, для работы с путями/аргументами).

## Критерии приемки

- [ ] Запуск из CLI с путями к одному или нескольким файлам.
- [ ] Поддержка типичных аудио и видео форматов (через извлечение/конвертацию аудио).
- [ ] Для каждого входного файла создаются два выходных: `.txt` и `.json` с таймкодами.
- [ ] В JSON есть массив сегментов с полями `start`, `end`, `text`.
- [ ] Работа полностью локальная (офлайн), язык — русский.
- [ ] README с инструкцией по установке (Python, ffmpeg, модель) и примерами запуска.

## Шаги реализации

### Шаг 1: Окружение и извлечение аудио
- **Сделать:** venv, requirements.txt (faster-whisper и т.д.), проверка наличия ffmpeg; функция извлечения аудио из файла в временный wav/mono для whisper.
- **Проверить:** из видео получается аудиофайл, который открывается в whisper.

### Шаг 2: Транскрипция одним файлом
- **Сделать:** загрузка модели faster-whisper (например, base или small), вызов транскрипции с таймкодами, язык ru.
- **Проверить:** на тестовом аудио получаются сегменты с start/end/text.

### Шаг 3: Выходные форматы
- **Сделать:** запись .txt (склейка текста сегментов) и .json (структура: segments + метаданные).
- **Проверить:** файлы создаются, JSON валидный, времена в секундах.

### Шаг 4: CLI и несколько файлов
- **Сделать:** парсинг аргументов (список файлов, опционально `--output-dir`), цикл по файлам, единообразные имена выходных файлов.
- **Проверить:** передача нескольких файлов обрабатывается, выходы в указанной или рядом с исходным файлом.

### Шаг 5: Документация и репо
- **Сделать:** README (назначение, установка ffmpeg + Python, установка зависимостей, примеры), .gitignore, подготовка к новому репозиторию.
- **Проверить:** по README можно повторить установку и запуск.

## Риски и митигация

- **Большой размер модели / долгая обработка:** использовать модель small или base в MVP; в README указать требования к RAM и диску.
- **Отсутствие ffmpeg:** в README явно указать необходимость установки ffmpeg и проверку в коде с понятным сообщением при отсутствии.
- **Кодировки путей (Windows):** использовать пути как строки и нормализацию; при необходимости передавать короткие пути.
